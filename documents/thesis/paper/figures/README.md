# 図表作成ガイド

## 必要な図のリスト

### システム概要図
- **図1**: システム全体アーキテクチャ
- **図2**: データフロー図
- **図3**: モジュール間の関係図

### 技術詳細図
- **図4**: 地形解析フロー
- **図5**: A* 3Dアルゴリズムの動作
- **図6**: コスト関数の設計
- **図7**: 経路平滑化の例

### 実験結果図
- **図8**: Unityシミュレーション環境
- **図9**: 5つの地形シナリオ
- **図10**: 経路計画結果の比較
- **図11**: 性能評価結果
- **図12**: パラメータ感度分析

## 各図の詳細説明

### 図1: システム全体アーキテクチャ
**目的**: システムの全体構成を示す
**内容**:
- Unityシミュレーション環境
- ROS2システム
- 地形解析モジュール
- 3D経路計画モジュール
- ロボット制御

**作成方法**:
- Mermaid記法でシステム図を作成
- 各コンポーネントの関係を矢印で表現
- データの流れを明確に示す

### 図2: データフロー図
**目的**: データの処理過程を示す
**内容**:
- PointCloud2 → VoxelGrid3D
- 地形解析 → 経路計画
- 結果の可視化

**作成方法**:
- シーケンス図として作成
- 各ステップの処理時間を記載
- データサイズの変化を表示

### 図3: モジュール間の関係図
**目的**: モジュールの依存関係を示す
**内容**:
- TerrainAnalyzer
- PathPlanner3D
- 評価モジュール

**作成方法**:
- クラス図として作成
- 依存関係を矢印で表現
- インターフェースを明確に示す

### 図4: 地形解析フロー
**目的**: 地形解析の処理過程を示す
**内容**:
- 点群データの前処理
- ボクセルグリッド生成
- 地形分類
- 傾斜計算

**作成方法**:
- フローチャートとして作成
- 各ステップの詳細を記載
- パラメータの影響を表示

### 図5: A* 3Dアルゴリズムの動作
**目的**: A*アルゴリズムの動作原理を示す
**内容**:
- 26近傍探索
- ヒューリスティック計算
- 優先度キューの管理
- 経路の構築

**作成方法**:
- アニメーションGIFとして作成
- 各ステップの状態を表示
- 探索過程を可視化

### 図6: コスト関数の設計
**目的**: コスト関数の構成を示す
**内容**:
- 距離コスト
- 傾斜コスト（非線形）
- 障害物近接コスト
- 安定性コスト

**作成方法**:
- 数式とグラフを組み合わせ
- 各コストの重みを表示
- 実際の計算例を示す

### 図7: 経路平滑化の例
**目的**: 経路平滑化の効果を示す
**内容**:
- 平滑化前の経路
- 平滑化後の経路
- 平滑化手法の比較

**作成方法**:
- 3Dプロットとして作成
- 経路の品質指標を表示
- 平滑化パラメータの影響を示す

### 図8: Unityシミュレーション環境
**目的**: 実験環境を示す
**内容**:
- Unityの3D環境
- Bunkerロボット
- 地形の様子

**作成方法**:
- Unityのスクリーンショット
- 複数のアングルから撮影
- 環境の特徴を強調

### 図9: 5つの地形シナリオ
**目的**: 実験シナリオを示す
**内容**:
- FlatWithObstacles
- GentleSlope
- SteepSlope
- DenseObstacles
- MixedTerrain

**作成方法**:
- 各シナリオの3D表示
- 地形の特徴を色分け
- 障害物の配置を表示

### 図10: 経路計画結果の比較
**目的**: 提案手法の有効性を示す
**内容**:
- 2D経路計画 vs 3D経路計画
- 各シナリオでの結果
- 経路の品質比較

**作成方法**:
- 経路を3Dプロット
- 色分けで手法を区別
- 統計情報を併記

### 図11: 性能評価結果
**目的**: 定量的な性能を示す
**内容**:
- 経路長の比較
- 計算時間の比較
- 成功率の比較

**作成方法**:
- 棒グラフとして作成
- エラーバーを表示
- 統計的有意性を記載

### 図12: パラメータ感度分析
**目的**: パラメータの影響を示す
**内容**:
- コスト重みの影響
- ボクセルサイズの影響
- 傾斜閾値の影響

**作成方法**:
- ヒートマップとして作成
- 最適パラメータを強調
- トレードオフを表示

## 図表作成のベストプラクティス

### 1. 統一性
- 色使いの統一
- フォントの統一
- スタイルの統一

### 2. 可読性
- 適切なサイズ
- 明確なラベル
- 十分なコントラスト

### 3. 情報量
- 必要な情報のみ
- 冗長な情報の削除
- 重要なポイントの強調

### 4. 技術的精度
- 正確な数値
- 適切なスケール
- 単位の明記

## 作成ツール

### 図作成
- **Mermaid**: システム図、フローチャート
- **matplotlib**: グラフ、プロット
- **Unity**: 3D可視化
- **Inkscape**: ベクター図形
- **GIMP**: 画像編集

### 表作成
- **pandas**: データ処理
- **LaTeX**: 表の整形
- **Excel**: 簡単な表
- **Google Sheets**: 共同編集

- [ ] ラベルが読みやすい
- [ ] 単位が明記されている
- [ ] 凡例が適切
- [ ] 図表番号が正しい
- [ ] キャプションが適切
- [ ] 本文との整合性
