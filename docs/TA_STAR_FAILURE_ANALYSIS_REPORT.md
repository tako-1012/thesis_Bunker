## 💡 失敗原因の推定

### 1. スケーラビリティの問題

- **Category 1 (150×150)**: 100% 成功
- **Category 2 (250×250)**: 100% 成功
- **Category 3 (400×400)**: 66.7% 成功 ⚠️

→ マップサイズが**400×400**を超えると、TA*の計算量が急増。

### 2. TA*固有のオーバーヘッド

TA*は地形適応のために以下の追加処理を実施：
- 地形複雑度マップの事前計算
- 8種類の地形タイプ分類
- 動的なコスト関数選択
- キャッシュ機構

→ **大規模環境では、これらのオーバーヘッドが支配的になる**

### 3. メモリまたはノード展開数の限界

TA*が失敗した理由の可能性：
- メモリ消費の上限到達
- 探索ノード数の上限到達
- ヒューリスティック関数の不適切な設定
- Open/Closedリストのサイズ超過

### 4. サブカテゴリ3_2の特異性

**3_2で6/10件失敗** → このサブカテゴリには特有の難しさがある可能性：
- 特定の地形パターン
- スタート/ゴール位置の配置
- 障害物の密度

   - 計算量削減の余地あり
