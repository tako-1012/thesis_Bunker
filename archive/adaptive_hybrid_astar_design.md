# Adaptive Hybrid A* (AHA*) アルゴリズム
## 3D経路計画用の提案手法

---

## 📝 概要

**Adaptive Hybrid A* (AHA*)** は、3次元空間における自律移動ロボットの経路計画のために開発された新しいハイブリッドアルゴリズムです。既存手法（A*, Weighted A*, RRT*）の長所を組み合わせ、状況に応じて探索戦略を動的に切り替えることで、**高速性と最適性を両立**します。

---

## 🎯 研究背景と動機

### 既存手法の課題

| アルゴリズム | 長所 | 短所 |
|------------|------|------|
| **A*** | 最適経路保証 | 大規模空間で計算時間大 |
| **Dijkstra** | 確実性 | 非常に遅い |
| **Weighted A*** | 高速 | 最適性が犠牲 |
| **RRT*** | 高次元空間に強い | 経路品質が不安定 |

### AHA*の狙い

> **「探索の初期は高速に、終盤は最適に」**

大規模3D空間では、最初から最適解を求めると膨大な計算時間がかかります。AHA*は探索の進捗に応じて戦略を変えることで、実用的な時間で高品質な経路を生成します。

---

## 🔬 アルゴリズムの詳細

### 1. 3段階探索戦略

AHA*は探索を3つのフェーズに分割します：

```
┌────────────────────────────────────────────────────┐
│ Phase 1: 初期探索 (Initial Exploration)            │
│   目的: 大まかな経路候補を高速に発見              │
│   手法: 高ε Weighted A* (ε=3.0～5.0)            │
│   特徴: ランダムサンプリングで探索範囲を拡大      │
└────────────────────────────────────────────────────┘
           ↓ (進捗30%)
┌────────────────────────────────────────────────────┐
│ Phase 2: 洗練 (Refinement)                         │
│   目的: 経路候補を改善・精緻化                     │
│   手法: 中ε Weighted A* (ε=1.5)                  │
│   特徴: バランスの取れた探索と開発                 │
└────────────────────────────────────────────────────┘
           ↓ (進捗70%)
┌────────────────────────────────────────────────────┐
│ Phase 3: 最適化 (Optimization)                     │
│   目的: 最終的な最適経路を確定                     │
│   手法: 標準A* (ε=1.0)                            │
│   特徴: 完全な最適性保証                           │
└────────────────────────────────────────────────────┘
```

### 2. 適応的パラメータ調整

#### 地形複雑度分析
経路計画前に、スタートからゴールまでの経路上の地形を分析：

```python
complexity = f(obstacle_density, slope, path_distance)
```

- **低複雑度** (0.0～0.3): 平地、障害物少 → 低εで開始
- **中複雑度** (0.3～0.6): 傾斜あり → 標準εで開始
- **高複雑度** (0.6～1.0): 障害物密、急傾斜 → 高εで開始

#### 動的フェーズ遷移
探索の進捗と地形複雑度に応じて、フェーズ遷移のタイミングを調整：

```
遷移閾値 = base_threshold × (1 + complexity × 0.3)
```

複雑な地形では初期探索を長めに実施します。

### 3. 適応的近傍探索

各フェーズで探索する近傍ノードを動的に変更：

#### Phase 1: 粗い探索
```python
# 26近傍のうち70%のみ探索（ランダムサンプリング）
if random.random() > 0.7:
    skip_neighbor()
```

#### Phase 2: 標準探索
```python
# 完全な26近傍探索
explore_all_26_neighbors()
```

#### Phase 3: ゴール指向探索
```python
# 26近傍 + ゴール方向の追加サンプリング
explore_all_26_neighbors()
add_goal_biased_samples(scale=2, 3)
```

## 📊 理論的性能分析

### 計算複雑度

| フェーズ | 時間複雑度 | 空間複雑度 |
|---------|-----------|-----------|
| Phase 1 | O(b^d × 0.7) | O(b^d) |
| Phase 2 | O(b^d × ε) | O(b^d) |
| Phase 3 | O(b^d) | O(b^d) |

ここで、`b`は分岐係数、`d`は深さ、`ε`はヒューリスティック重み

### 最適性保証

- **Phase 3で標準A*を使用** → 最終経路は最適性を保証
- **Phase 1, 2は初期解の高速発見** → 実用的な時間で完了

### 期待される効果

1. **計算時間の短縮**: 初期段階での高速探索により、平均30～50%の時間短縮
2. **経路品質の維持**: 最終段階でA*を使用するため、最適性を損なわない
3. **適応性**: 地形に応じたパラメータ調整で、様々な環境に対応

## 🧪 実験設計

### 比較対象アルゴリズム

1. **A*** - 標準最適経路計画
2. **Dijkstra** - 全探索法
3. **Weighted A* (ε=1.5)** - 高速化版
4. **RRT*** - ランダムサンプリング法
5. **AHA* (提案手法)** ← これを評価

### 評価指標

| 指標 | 説明 | 重要度 |
|-----|------|--------|
| **計算時間** | 経路計画にかかった時間 [秒] | ⭐⭐⭐⭐⭐ |
| **探索ノード数** | 探索したノードの総数 | ⭐⭐⭐⭐ |
| **経路長** | 生成された経路の総距離 [m] | ⭐⭐⭐⭐⭐ |
| **成功率** | 有効な経路を発見した割合 [%] | ⭐⭐⭐⭐⭐ |
| **地形適応性** | 複雑地形での性能 | ⭐⭐⭐ |

### テストシナリオ

16種類のシナリオで評価：

#### カテゴリ1: 平地
- 短距離（5m）
- 中距離（10m）
- 長距離（20m）

#### カテゴリ2: 傾斜
- 緩傾斜（10度）
- 中傾斜（20度）
- 急傾斜（25度）

#### カテゴリ3: 複雑地形
- 障害物あり
- 3D複雑経路
- 負の座標を含む
