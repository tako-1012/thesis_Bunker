# デモシナリオと台本

## デモ概要
不整地環境における3D経路計画システムの実演デモ

## デモ環境
- **Unityシミュレーション**: 5つの地形シナリオ
- **ROS2システム**: リアルタイム処理
- **Bunkerロボット**: 実機モデル

---

## デモシナリオ1: 平坦地形での基本動作

### シナリオ設定
- **地形**: FlatWithObstacles
- **障害物**: 流木、岩、木
- **目標**: 基本的な障害物回避

### デモ手順
1. **システム起動**
   ```bash
   # Unityシミュレーション起動
   ros2 launch bunker_gazebo bunker_world.launch.py
   
   # RTAB-Map起動
   ros2 launch bunker_nav kinect_rtabmap.launch.py
   
   # 3D経路計画システム起動
   ros2 launch bunker_3d_nav full_system.launch.py
   ```

2. **地形解析の確認**
   - Rvizで点群データの表示
   - ボクセルグリッドの可視化
   - 地面・障害物の分類結果

3. **経路計画の実行**
   - スタート地点の設定
   - ゴール地点の設定
   - 3D経路の生成

4. **結果の確認**
   - 生成された経路の表示
   - 経路コストの表示
   - 計算時間の確認

### 台本
**発表者**: 「まず、平坦地形での基本動作を確認します。この地形には流木や岩などの障害物が配置されています。」

**システム起動**
**発表者**: 「Unityシミュレーション、RTAB-Map、3D経路計画システムを起動します。」

**地形解析の確認**
**発表者**: 「RTAB-Mapが取得した3D点群データを、地形解析モジュールが処理しています。赤い部分が障害物、緑の部分が地面として分類されています。」

**経路計画の実行**
**発表者**: 「スタート地点とゴール地点を設定して、3D経路計画を実行します。」

**結果の確認**
**発表者**: 「青い線が生成された3D経路です。障害物を適切に回避し、安全な経路が計画されています。計算時間は0.8秒、経路長は5.8メートルです。」

## デモシナリオ2: 傾斜地形での安全性評価

### シナリオ設定
- **地形**: SteepSlope
- **傾斜角度**: 20-30度
- **目標**: 傾斜を考慮した安全な経路

### デモ手順
1. **地形の確認**
   - 傾斜角度の表示
   - 安定性マップの可視化
   - 走行可能性の判定

2. **経路計画の実行**
   - 2D経路計画との比較
   - 3D経路計画の実行
   - 傾斜コストの表示

3. **安全性の評価**
   - 最大傾斜角の確認
   - 転倒リスクの評価
   - 安定性スコアの表示

### 台本
**発表者**: 「次に、傾斜地形での安全性評価を行います。この地形は20-30度の急傾斜があります。」

**地形の確認**
**発表者**: 「地形解析モジュールが傾斜角度を計算しています。色分けで傾斜の度合いが表示されています。赤い部分は危険な傾斜、緑の部分は安全な傾斜です。」

**経路計画の実行**
**発表者**: 「2D経路計画では、傾斜を考慮せずに最短経路を選択します。一方、3D経路計画では、傾斜を考慮した安全な経路を選択します。」

**安全性の評価**
**発表者**: 「3D経路計画では、最大傾斜角が25度に抑制され、転倒リスクが大幅に低減されています。安全性が向上していることが確認できます。」

## デモシナリオ4: リアルタイム性能の確認

   - CPU使用率の確認

2. **動的環境での対応**
   - 環境変化への対応
   - 経路の再計画
   - 適応性の確認

### 台本
**発表者**: 「リアルタイム性能を確認します。処理時間、メモリ使用量、CPU使用率を監視しています。」

**リアルタイム処理の確認**

**動的環境での対応**
