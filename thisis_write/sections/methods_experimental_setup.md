## 3.3 実験条件

本節では、96シナリオによる総合評価実験の詳細な条件を記述する。

### 3.3.1 シナリオ構成

実験では、環境の複雑度を系統的に変化させた96個のテストシナリオを使用した。シナリオは地形サイズと複雑度に基づいて以下のように分類される。

#### マップサイズの分類

| カテゴリ | サイズ | シナリオ数 | 割合 |
|---------|--------|-----------|------|
| SMALL   | 50×50  | 30        | 31.3% |
| MEDIUM  | 100×100| 48        | 50.0% |
| LARGE   | 200×200| 18        | 18.8% |
| **合計** | -      | **96**    | **100%** |

#### 地形複雑度の特徴

各シナリオには以下の地形特性が含まれる：

- **障害物密度**: 5%〜40%
- **地形傾斜**: 0度（平地）〜30度（急傾斜）
- **地形変動**: 単調な平地から複雑な起伏地形
- **ランダムシード**: 固定値（再現性確保）

複雑度の分類：
- **Simple（単純）**: 障害物密度 < 15%、傾斜 < 10度
- **Medium（中程度）**: 障害物密度 15-25%、傾斜 10-20度
- **Complex（複雑）**: 障害物密度 > 25%、傾斜 > 20度

### 3.3.2 アルゴリズムのパラメータ設定

実験で評価した4つのアルゴリズムのパラメータ設定を以下に示す。

#### TA* (Terrain-Aware A*)

提案手法TA*のパラメータ設定：

| パラメータ | 値 | 説明 |
|-----------|-----|------|
| `terrain_weight` | 0.3 | 地形コストの重み係数 |
| `heuristic_multiplier` | 1.5 | ヒューリスティック関数の乗数 |
| `prune_distance_factor` | 2.0 | プルーニング距離係数 |
| `cost_limit_factor` | 3.0 | コスト上限係数 |
| `timeout` | 120秒 | 最大計算時間制限 |

**設計根拠**:
- `terrain_weight = 0.3`: 地形の影響を考慮しつつ、過度な重み付けを避ける
- `heuristic_multiplier = 1.5`: 楽観的すぎない推定により確実性を優先
- `prune_distance_factor = 2.0`: 探索空間の効率的な削減
- タイムアウト120秒: 極端に複雑なシナリオでの計算時間制限

#### AHA* (Anytime Hierarchical A*)

| パラメータ | 値 | 説明 |
|-----------|-----|------|
| `hierarchy_levels` | 3 | 階層レベル数 |
| `anytime_iterations` | 5 | 反復回数 |
| `heuristic_weight` | 1.0 | ヒューリスティック重み |

#### Theta*

| パラメータ | 値 | 説明 |
|-----------|-----|------|
| `line_of_sight_check` | True | 視線チェックの有効化 |
| `heuristic_weight` | 1.0 | ヒューリスティック重み |

#### FieldD*Hybrid

| パラメータ | 値 | 説明 |
|-----------|-----|------|
| `base_planner` | D* Lite | 基本経路計画手法 |
| `window_size` | 5×5 | スライディングウィンドウサイズ |
| `local_iterations` | 3 | 局所最適化反復回数 |

#### 実験プロトコル

各シナリオに対して：

1. **初期化**: シナリオファイルからマップを読み込み
2. **経路計画実行**: 指定アルゴリズムで経路を計算
3. **計測**: 計算時間、経路長、探索ノード数を記録
4. **検証**: 経路の妥当性を確認（衝突判定、連結性）
5. **記録**: 結果をJSON形式で保存

各シナリオは**3回実行**し、中央値を採用した（計算時間の変動を抑制）。

### 3.3.4 統計解析手法

#### 記述統計

各アルゴリズムについて以下を計算：

- **平均値（Mean）**: 計算時間の平均
- **標準偏差（SD）**: データのばらつき
- **中央値（Median）**: 分布の中心傾向（外れ値の影響を受けにくい）
- **四分位範囲（IQR）**: Q3 - Q1、分布の広がり
- **95%信頼区間**: $\text{Mean} \pm t_{\alpha/2, n-1} \times \text{SE}$

ここで、$\text{SE} = \text{SD} / \sqrt{n}$ は標準誤差、$t_{\alpha/2, n-1}$ はt分布の臨界値（自由度 $n-1$、有意水準 $\alpha=0.05$）。

#### 推測統計

**Welch t検定（Welch's t-test）**:

2群の平均値の差を検定するため、Welch t検定を使用した。この手法は不等分散を仮定し、標準的なStudent t検定よりも頑健である。

$$
t = \frac{\bar{X}_1 - \bar{X}_2}{\sqrt{\frac{s_1^2}{n_1} + \frac{s_2^2}{n_2}}}
$$

ここで、$\bar{X}_i$ は平均、$s_i^2$ は分散、$n_i$ はサンプルサイズ（群 $i = 1, 2$）。

自由度は以下のWelch-Satterthwaite式で計算：

$$
\text{df} = \frac{\left(\frac{s_1^2}{n_1} + \frac{s_2^2}{n_2}\right)^2}{\frac{(s_1^2/n_1)^2}{n_1-1} + \frac{(s_2^2/n_2)^2}{n_2-1}}
$$

**有意水準**: $\alpha = 0.05$（両側検定）

**効果量（Cohen's d）**:

統計的有意性だけでなく、実務的な効果の大きさを評価するため、Cohen's dを計算した。

$$
d = \frac{\bar{X}_1 - \bar{X}_2}{s_{\text{pooled}}}
$$

ここで、プールされた標準偏差 $s_{\text{pooled}}$ は：

$$
s_{\text{pooled}} = \sqrt{\frac{(n_1 - 1)s_1^2 + (n_2 - 1)s_2^2}{n_1 + n_2 - 2}}
$$

**効果量の解釈**（Cohen, 1988）:
- $|d| < 0.2$: 無視できる効果
- $0.2 \leq |d| < 0.5$: 小さい効果
- $0.5 \leq |d| < 0.8$: 中程度の効果
- $|d| \geq 0.8$: 大きい効果

#### 検出力分析

サンプルサイズ $n=96$ の妥当性を検証するため、事後的検出力分析を実施した。

**条件**:
- 有意水準 $\alpha = 0.05$
- 検出力 $1-\beta = 0.80$（80%）
- 期待効果量 $d = 0.8$（大きい効果）

**必要サンプルサイズ**: 約30（各群）

**実際のサンプルサイズ**: 96（各群）

結論: 本実験のサンプルサイズは統計的に十分な検出力を持つ。

### 3.3.5 データ管理と再現性

#### データ保存形式

すべての実験結果はJSON形式で保存された：

```json
{
  "scenario_id": "SMALL_1",
  "results": {
    "TA*": {
      "computation_time": 0.41,
      "path_length": 5.15,
      "nodes_explored": 40,
      "success": true
    },
    ...
  }
}
```

#### 再現性の確保

- **ランダムシード固定**: すべてのシナリオで同一のシード値を使用
- **パラメータ記録**: すべてのアルゴリズムパラメータを記録
- **バージョン管理**: ソフトウェアバージョンを明記
- **実験ログ**: タイムスタンプ付きで詳細ログを保存

### 3.3.6 評価指標

#### 主要指標

1. **計算時間（Computation Time）**: 経路計画の実行時間（秒）
2. **成功率（Success Rate）**: 有効な経路を発見した割合（%）
3. **経路長（Path Length）**: 発見された経路の長さ（メートル）
4. **探索ノード数（Nodes Explored）**: 探索したノードの数

#### 統計的比較

すべてのアルゴリズムペアに対して：
- Welch t検定による有意性検定
- Cohen's dによる効果量評価
- 95%信頼区間による推定精度評価
