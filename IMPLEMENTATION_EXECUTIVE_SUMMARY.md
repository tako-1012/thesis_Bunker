# アルゴリズム実装の正確性: エグゼクティブサマリー

**作成日**: 2026年1月29日  
**質問**: 「アルゴリズムは正しくプログラムに実装されてるか？」  
**回答**: ⚠️ **基本的には正しいが、3つの重大な問題がある**

---

## 【最重要な発見】

### 1️⃣ **TA*のプルーニングが過度に厳しい 🔴 重大**

```
問題: 迂回経路が探索から除外される可能性
原因: prune_distance_factor = 2.0 が固定

例: 障害物を避けるため遠回りする経路
    直線距離 100ボクセル → 許可: 200ボクセルまで
    だが、迂回して 220 ボクセルなら棄却される!
```

**影響**: 経路が見つかっても、本来見つかるはずの経路を逃す可能性

---

### 2️⃣ **地形コスト重み付けが弱すぎる 🔴 重大**

```
TA*の実装:
  adjusted_terrain_cost = 1.0 + 0.3 × (terrain_cost - 1.0)
  
例: 急勾配で terrain_cost = 3.0 なら
  adjusted_cost = 1.0 + 0.3 × 2.0 = 1.6
  
つまり、60%のコスト軽減 = 地形ペナルティが弱い
```

**影響**: 急勾配でも大きなペナルティを受けない → 「地形適応」が不十分

---

### 3️⃣ **A*が地形コストを完全に無視している 🟡 既知**

```
A*の実装:
  def _get_edge_cost(self, from_pos, to_pos, voxel_grid, cost_function):
      distance = self._heuristic(from_pos, to_pos)
      return distance * self.voxel_size  # ← cost_function を使わない!
```

**影響**: A* と TA* の比較が不公平 (A*が有利)

---

| 項目 | スコア | 理由 |

| **基本的なA*アルゴリズム** | ⭐⭐⭐⭐⭐ | Open/Closed Set管理は正確 |
| **26近傍探索** | ⭐⭐⭐⭐⭐ | 3D探索ロジックは正確 |
| **経路復元** | ⭐⭐⭐⭐⭐ | parent[]辞書での復元は正確 |
| **地形コスト統合** | ⭐⭐⭐☆☆ | パラメータが最適化されていない |
| **プルーニング戦略** | ⭐⭐⭐☆☆ | 過度に厳しい可能性 |
| **ヒューリスティック関数** | ⭐⭐⭐⭐☆ | 1.5倍乗数が妥当か不明 |
| **比較の公平性** | ⭐⭐☆☆☆ | A*が地形を無視のまま |

## 【論文への影響】

### ✅ 信頼できる結果

- TA* vs RRT*: **19.7倍高速** ← 確実  
- TA*成功率: **100%** ← 確実（ただし理由が不明確）

### ❌ 疑わしい結果

- TA* パラメータが最適値か不明
- A*が公平に評価されているか不明  
- プルーニングの有効性が測定されていない

## 【修正が必須な3つのコード】

### 修正1: プルーニング距離を緩和

```python
# 現在
if dist_to_goal_raw > straight_distance * 2.0:  continue

# 修正後
if dist_to_goal_raw > straight_distance * 2.5:  continue
# または、プルーニングを除去
```

### 修正2: 地形重み付けを強化

```python
# 現在
terrain_weight = 0.3

# 修正後 (より強い地形反映)
terrain_weight = 0.7 または 1.0
# または、感度分析で最適値を決定
```

### 修正3: A*に地形コスト機能を追加

```python
# 現在 (A*の_get_edge_cost)
return distance * self.voxel_size  # 地形無視

# 修正後
if cost_function is not None:
    terrain_cost = cost_function(from_pos, to_pos)
    return distance * self.voxel_size * terrain_cost
```
